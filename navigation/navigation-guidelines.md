# Sidebar & Page Navigation Guidelines

A clear structure ensures users find what they need instantly. GitBook addresses this using a highly scannable, two-tier navigation map.

## Left-Side Navigation Rules
The left sidebar handles global site navigation.
* Use category headings in `SUMMARY.md` to split major concepts.
* Use indentation to nest sub-pages under parent topics to keep the sidebar tight.

## Right-Side Navigation Rules
The right-side navigation is completely auto-generated by GitBook by reading the heading structures built directly into the active Markdown page.
* **H1 (`#`)**: Reserve this strictly for the main page title at the very top.
* **H2 (`##`)**: Generates primary targets in the right-side sub-navigation layout.
* **H3 (`###`)**: Generates minor indented targets beneath the main H2 markers on the right side.

## Documentation Platform Comparison
Review the table below to see how structured GitBook spaces outperform standard text README files:

| Feature Overview | GitBook Integrated | Standard GitHub README |
| :--- | :--- | :--- |
| **Left Sidebar Map** | Multi-level nested menus | Not available (Flat file view) |
| **Right Sub-Navigation** | Auto-generated from page headings | Requires manual HTML anchor tags |
| **Global Live Search** | Indexed instantly across all pages | Limited to browser text search (Ctrl+F) |
| **Visual Elements** | Native tables, callouts, and hints | Basic, unstyled plaintext Markdown |
