# Creating Your First Space

A GitBook Space represents an isolated, single public documentation project instance linked directly to a tracking context.

## Initializing the Space Container
Follow this sequence to instantiate a target workspace context:

### 1. Allocation Sequence
From your core dashboard environment, click the **New Space** button. This creates an unconfigured, completely clean documentation silo container.

### 2. Space Naming Conventions
Provide a logical, clear title matching your source code repository intent. For this guide, name the space `My GitBook Docs Framework`.
