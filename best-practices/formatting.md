# Syncing & Formatting Best Practices

Once your file architecture is locked in, use these clean workflows to avoid sync breaks and formatting errors.

## Triggering an Automatic Sync
Whenever you push code adjustments to your default GitHub branch, GitBook hooks detect the changes and rebuild your public documentation site automatically within a few seconds.

## Formatting Reminders
* Always make sure file paths in your `SUMMARY.md` match your exact folder case-sensitivity.
* Use Markdown comparison tables to summarize information for quick user consumption.
* Avoid spaces or heavy special characters in filenames to prevent broken link generation.
